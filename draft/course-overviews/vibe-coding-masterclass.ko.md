# 바이브 코딩 실전 마스터클래스

## 강좌 개요

이 강좌는 생성형 AI를 “코드를 대신 써 주는 도구”로만 보는 대신, 작은 제품을 빠르게 설계하고 구현하고 검증하는 작업 시스템으로 다루는 방법을 설명합니다. 초급자가 가장 많이 막히는 지점은 프롬프트 자체보다도 무엇을 먼저 정리해야 하는지, 어느 순간에 문서를 써야 하는지, 어디까지 AI에게 맡기고 어디서 사람이 기준을 잡아야 하는지를 모른다는 데 있습니다. 이 강좌는 그 순서를 분해해서 보여 줍니다.

수강생은 아이디어를 MVP 수준으로 압축하고, PRD와 AGENTS 문서로 작업 맥락을 분리하고, 과한 기술 선택을 피하면서도 실제로 배포 가능한 구조를 잡는 흐름을 배우게 됩니다. 강좌의 목적은 “멋진 한 번의 프롬프트”를 만드는 것이 아니라, 반복 가능한 개인 작업 루프를 세우는 데 있습니다.

강의를 끝까지 따라가면 작은 서비스 아이디어를 기준으로 문서화, 스택 선정, 아키텍처 스케치, 검증 계획, 배포 전 점검까지 한 번에 이어지는 MVP 제작 흐름을 스스로 설명할 수 있게 됩니다. 예제 자료는 템플릿과 구조 예시에 집중되어 있어서, 초급자도 큰 코드베이스 없이 바로 자기 프로젝트에 적용해 볼 수 있습니다.

## 추천 수강생

- AI 코딩 도구를 써 보긴 했지만 프로젝트 전체 흐름으로 연결하지 못한 입문 개발자
- 혼자 빠르게 MVP를 만들고 검증해야 하는 대학생, 1인 창업가, 사이드 프로젝트 제작자
- 프롬프트 문장보다 문서, 작업 단위, 검증 루프가 더 중요하다는 감각을 배우고 싶은 수강생

## 사전 준비

- HTML, CSS, 자바스크립트 또는 React 기초가 있으면 이해가 가장 빠릅니다.
- AI 코딩 도구를 처음 써도 괜찮지만, 파일과 폴더 구조는 읽을 수 있어야 합니다.
- 문서 편집기와 간단한 배포 플랫폼 개념 정도를 알고 있으면 실습에 도움이 됩니다.

## 학습 목표

- PRD와 AGENTS 문서를 이용해 AI에게 안정적인 작업 맥락을 전달할 수 있습니다.
- 프로젝트 성격에 맞는 기술 스택과 비용 범위를 현실적으로 좁힐 수 있습니다.
- 구현, UI 정리, 테스트, 배포를 따로 떼지 않고 하나의 반복 루프로 연결할 수 있습니다.
- MVP를 공개하기 전에 확인해야 하는 최소 보안 및 검증 체크리스트를 만들 수 있습니다.

## 상세 커리큘럼

### 1. 철학과 작업 루프 이해

첫 챕터에서는 바이브 코딩을 단순한 코드 생성 기법이 아니라 작업 시스템으로 보는 관점을 잡습니다. 목표를 어떻게 정의해야 AI가 흔들리지 않는지, 철학과 원칙을 왜 먼저 정리해야 하는지, 그리고 결과물을 어떤 형태로 쪼개야 이후 단계가 쉬워지는지를 설명합니다.

이 챕터에서 수강생은 “무엇을 만들 것인가”보다 “어떻게 반복 가능한 방식으로 만들 것인가”를 먼저 생각하는 습관을 배우게 됩니다. 강의는 가드레일이 있는 작업 루프를 소개하면서, 사람이 기준을 세우고 AI는 그 기준 아래에서 속도를 높이는 구조가 왜 중요한지 보여 줍니다.

### 2. 문서와 컨텍스트 준비

두 번째 챕터는 AGENTS.md와 PRD의 역할을 나눠 보는 시간입니다. 운영 규칙 문서와 제품 요구 문서를 분리하지 않으면 AI가 쉽게 혼동한다는 점을 실례 중심으로 설명하고, 초급자도 바로 가져다 쓸 수 있는 문서 템플릿 구조를 함께 다룹니다.

수강생은 이 챕터를 통해 “작업 규칙은 어디에 적고, 제품 요구는 어디에 적는가”를 명확하게 구분할 수 있게 됩니다. 예제 AGENTS 파일과 PRD 템플릿을 통해, 큰 프로젝트가 아니어도 문서화가 왜 개발 속도를 올리는지 직접 확인할 수 있습니다.

### 3. 기술 스택과 비용 결정

세 번째 챕터에서는 프로젝트 유형에 따라 어떤 기술 스택이 과한지, 어떤 선택이 MVP에 적당한지 판단하는 방법을 배웁니다. Skills와 MCP 같은 도구 선택의 차이, 프런트엔드와 백엔드의 최소 조합, 그리고 월 비용을 어디까지 허용할 것인지 같은 현실적인 판단 기준을 다룹니다.

핵심은 정답 스택을 외우는 것이 아니라, “이번 프로젝트가 반드시 증명해야 하는 한 가지”를 기준으로 가장 단순한 조합을 고르는 법을 익히는 것입니다. 수강생은 강의를 마친 뒤 스택 결정표를 이용해 자기 프로젝트 후보군을 빠르게 줄일 수 있어야 합니다.

### 4. MVP 아키텍처와 구현 흐름

네 번째 챕터는 식당 주문 MVP 예제를 중심으로, 화면 계층과 API 계층, 데이터 계층을 어떻게 나눌지 설명합니다. 초급자에게 중요한 것은 복잡한 마이크로서비스가 아니라, AI에게 설명하기 쉽고 사람이 직접 검증하기 쉬운 구조를 잡는 일이라는 점을 강조합니다.

이 과정에서 수강생은 구현 단위를 얼마나 잘게 나누어야 AI와 협업하기 좋은지, 어떤 경계가 있으면 테스트와 수정이 쉬워지는지 배웁니다. 예제 아키텍처 문서는 나중에 자기 아이디어를 비슷한 형태로 옮겨 적는 출발점이 됩니다.

### 5. 검증, 보안, 배포

마지막 챕터에서는 “동작하는 MVP”를 “내놓을 수 있는 MVP”로 바꾸는 마무리 단계를 다룹니다. 무엇을 먼저 테스트해야 하는지, 배포 전에 최소한 어떤 보안 항목을 확인해야 하는지, 그리고 빠른 배포와 무모한 배포의 차이가 무엇인지 정리합니다.

이 챕터를 통해 수강생은 AI가 만든 결과물을 그대로 믿는 대신, 핵심 시나리오를 사람이 직접 검증하고 배포 직전 체크리스트로 리스크를 줄이는 습관을 배우게 됩니다. 작은 MVP일수록 검증 기준이 더 중요하다는 메시지가 강좌 전체를 마무리합니다.

## 실습 자료와 산출물

강좌에서 사용하는 예제 자료는 실제 프로젝트를 통째로 제공하는 대신, 반복 사용 가능한 문서와 구조 템플릿에 집중되어 있습니다. 예제 README는 전체 자료를 어떤 순서로 읽어야 하는지 안내하고, AGENTS 예시는 운영 규칙을 AI에게 전달하는 방식을 보여 줍니다. PRD 템플릿은 아이디어를 구현 가능한 요구사항으로 바꾸는 출발점 역할을 합니다.

또한 스택 결정표와 MVP 아키텍처 예시는 초급자가 자기 프로젝트에 그대로 변형 적용할 수 있는 사고 틀을 제공합니다. 수강 후에는 개인 작업 루프, 스택 결정 메모, 아키텍처 초안, 배포 전 검증 메모 같은 산출물을 직접 남기는 것을 목표로 합니다.

## 추천 학습 방법

가장 좋은 순서는 PRD 템플릿과 AGENTS 예시를 먼저 읽고, 그다음 슬라이드를 따라가며 왜 그런 문서가 필요한지 이해하는 것입니다. 이후 스택 결정표와 아키텍처 예시를 보면서 자기 아이디어를 한 번 적어 보면 강의 내용이 훨씬 빠르게 자기 것이 됩니다.

실습은 아이디어 선정, 문서화, 스택 결정, 아키텍처 작성, 검증 계획 수립 순서로 그대로 따라가는 것이 좋습니다. 처음부터 완성도 높은 제품을 만들려고 하기보다, 작은 범위에서 끝까지 가 보는 경험을 만드는 쪽이 이 강좌의 목적에 더 가깝습니다.

## FAQ

### 이 강좌는 특정 AI 도구에만 묶여 있나요?

출발점은 Claude Code이지만, 핵심은 문서화와 검증 루프입니다. 따라서 Cursor, Codex, 다른 AI 코딩 환경으로도 상당 부분 옮겨 갈 수 있습니다. 중요한 것은 도구 이름보다도 작업 구조입니다.

### 초급자가 바로 MVP를 배포해도 되나요?

무조건 빠르게 배포하는 것이 목표는 아닙니다. 이 강좌는 빨리 만들되, 무엇을 직접 검증하고 어떤 가드레일을 세워야 하는지 함께 배우는 데 초점을 둡니다. 작은 범위에서 안전하게 공개하는 법을 익히는 것이 더 중요합니다.

### 예제 코드가 큰 프로젝트인가요?

아닙니다. 초급자에게 필요한 것은 거대한 코드베이스보다 재사용 가능한 문서와 간단한 구조 예시입니다. 그래서 강좌는 템플릿과 의사결정 프레임워크 중심으로 구성되어 있습니다.
