# Cusdis 자동화 운영 체크리스트

## 워크플로우를 켜기 전

- webhook URL이 HTTPS로 정상 응답하는지 확인합니다.
- 정상 댓글 1개와 스팸성 댓글 1개를 각각 테스트합니다.
- Gemini 자격 증명과 모델 사용량 한도를 확인합니다.
- If 노드가 `NORMAL` 외 결과를 차단하는지 확인합니다.
- HTTP Request 노드가 올바른 사이트와 인증값을 쓰는지 점검합니다.

## 배포 후

- 처음 5개의 댓글은 직접 눈으로 확인합니다.
- JSON 파싱 실패 로그를 별도로 모아 둡니다.
- 필요 시 워크플로우를 끄고 수동 승인으로 되돌릴 수 있는 경로를 확보합니다.
- 오탐/미탐을 주간 단위로 검토하고 프롬프트 계약을 조금씩 조정합니다.
