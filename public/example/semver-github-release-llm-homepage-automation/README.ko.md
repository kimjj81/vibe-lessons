# SemVer Release 자동화 예제

이 폴더는 강좌 상세강의자료에서 연결되는 실습 자산을 모아 둔 곳입니다.

## 추천 순서

1. `release-pipeline-map.ko.md`
2. `release-workflow-checklist.ko.md`
3. `polish-release-notes-prompt.ko.md`
4. `homepage-sync-checklist.ko.md`

## 이 파일들로 보는 것

- 전체 release 파이프라인이 어떤 단계로 나뉘는지 먼저 확인합니다.
- release workflow 체크리스트로 태그 정책과 draft release 전략을 점검합니다.
- polishing 프롬프트로 LLM을 후처리 계층으로 제한하는 방식을 봅니다.
- homepage sync 체크리스트로 dispatch, import, build, no-op 재실행까지 연결합니다.

source repo와 homepage repo를 동시에 펼쳐 두고 읽는 것이 가장 좋습니다. 먼저 권한과 책임 경계를 고정한 다음, generated notes와 검증 순서를 추가하세요.
