# SemVer Release Automation Examples

This folder contains the practical assets linked from the Lecture Guide.

## Recommended order

1. `release-pipeline-map.md`
2. `release-workflow-checklist.md`
3. `polish-release-notes-prompt.md`
4. `homepage-sync-checklist.md`

## What these files are for

- The pipeline map gives you the full release path first.
- The workflow checklist clarifies tag policy and the draft-release strategy.
- The polishing prompt shows how the LLM is constrained as a post-processing layer.
- The homepage sync checklist ties together dispatch, import, build, and no-op reruns.

## Suggested exercise

Keep both the source repo and the homepage repo open while reading. Lock the ownership boundary first, then add generated notes and verification rules on top of it.
