# 스택 결정표

| 프로젝트 형태 | 프런트엔드 | 백엔드 / 데이터 | 이유 |
| --- | --- | --- | --- |
| 랜딩 페이지 + 폼 | Astro 또는 Next.js | 폼 백엔드 / 서버리스 | 빠른 출시, 적은 구성요소 |
| 대시보드 MVP | Next.js | Supabase | 인증과 데이터 작업 속도가 좋음 |
| 콘텐츠 중심 서비스 | Next.js 또는 Astro | Headless CMS | 편집 흐름이 중요함 |
| 자동화 중심 제품 | React 또는 Next.js | Worker / queue / database | 페이지 수보다 비동기 작업이 중요함 |

## 기본 원칙

MVP가 반드시 증명해야 하는 한 가지를 지키면서 가장 단순한 스택을 선택합니다.
